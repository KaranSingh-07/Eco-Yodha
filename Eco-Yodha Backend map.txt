PART 3: DATABASE MODELS (MongoDB/Mongoose)
3.1 User
// js
{
  _id, name, email, password (hashed),
  role: "student" | "teacher",
  // Student fields:
  school, grade, section, classroomCode,
  // Teacher fields:
  institutionName, department, phone, location,
  specialization, classrooms: [ObjectId],
  // Gamification:
  level: Number, xp: Number,
  currentStreak: Number, lastActiveDate: Date,
  avatar: String (initials),
  createdAt, updatedAt
}
3.2 Classroom
// js
{
  _id, name (e.g. "10-A"),
  code: String (unique 6-digit),
  teacher: ObjectId (ref User),
  students: [ObjectId (ref User)],
  createdAt
}
3.3 Module (Learning Path)
// js
{
  _id, title, type: "video" | "quiz" | "major-quiz" | "quest",
  description, videoUrl,
  position: Number (order in path),
  order: Number (difficulty tier),
  ecoPointsReward: Number,
  createdAt
}
3.4 Quiz
// js
{
  _id, module: ObjectId (ref Module),
  questions: [{
    question: String,
    options: [String],
    correctAnswer: Number (index),
    explanation: String
  }]
}
3.5 Quest (Community Task)
// js
{
  _id, title, region, description,
  status: "active" | "locked",
  points: Number, doublePoints: Boolean,
  deadline: Date,
  badges: [ObjectId (ref Badge)],
  requiredLevel: Number,
  createdAt
}
3.6 Badge
// js
{
  _id, name, description,
  rarity: "common" | "rare" | "epic" | "legendary",
  icon: String, criteria: String
}
3.7 UserProgress
// js
{
  _id, user: ObjectId (ref User),
  completedModules: [ObjectId],
  currentModule: ObjectId,
  carbonScore: Number, waterScore: Number, soilScore: Number,
  totalEcoPoints: Number,
  weeklyPoints: Number,
  streak: Number, lastActiveDate: Date,
  updatedAt
}
3.8 QuizSubmission
// js
{
  _id, user: ObjectId (ref User),
  quiz: ObjectId (ref Quiz),
  module: ObjectId (ref Module),
  answers: [Number], score: Number,
  totalQuestions: Number,
  ecoPointsEarned: Number,
  submittedAt: Date
}
3.9 QuestSubmission
// js
{
  _id, user: ObjectId (ref User),
  quest: ObjectId (ref Quest),
  files: [{ url: String, filename: String }],
  status: "pending" | "approved" | "rejected",
  teacherComment: String,
  verifiedBy: ObjectId (ref User),
  submittedAt: Date, verifiedAt: Date
}
3.10 UserBadge
// js
{
  _id, user: ObjectId (ref User),
  badge: ObjectId (ref Badge),
  earnedAt: Date
}

PART 6: HOW FRONTEND ↔ BACKEND CONNECT
Frontend (Vite :5173)  ←→  Backend (Express :5000)
        ↓                          ↓
   React components         API endpoints
        ↓                          ↓
   src/services/api.js     routes → controllers → models → MongoDB
        ↓
   AuthContext (JWT stored in localStorage)
        ↓
   Protected routes redirect to / if no token
Frontend additions needed:
-  src/contexts/AuthContext.tsx  — provides user state + token
-  src/services/api.ts  — axios instance with base URL + auth header interceptor
-  src/components/ProtectedRoute.tsx  — redirects unauthenticated users
-  src/hooks/useApi.ts  — reusable fetch wrapper with loading/error states
